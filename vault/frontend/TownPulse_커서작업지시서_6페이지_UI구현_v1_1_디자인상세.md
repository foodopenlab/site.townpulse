# TownPulse — Cursor 작업지시서 (6페이지 UI 구현 / 디자인 상세 포함)

> 작성일: 2026-06-23  
> 프로젝트명: **TownPulse**  
> 팀 성격: **해커톤 출품 준비 팀**  
> 전달 대상: **Cursor AI 개발 에이전트**  
> 구현 범위: **데스크톱 우선 6페이지 + 모바일 반응형 대응**  
> 디자인 기준: **샘플 2 방향 채택 + 와이어프레임/레이아웃/디자인 시스템 상세 명시**

---

# 0. 이번 문서의 목적

이 문서는 단순 요구사항 목록이 아니다.  
**Cursor가 바로 화면을 구현할 수 있도록, 각 페이지의 와이어프레임 구조 / 레이아웃 비율 / 카드 배치 / 타이포 위계 / 컬러 / 간격 / 컴포넌트 규칙까지 포함한 실전 지시서**다.

즉, 아래 내용을 보고 Cursor는 다음을 수행해야 한다.

1. 공통 디자인 시스템 구축
2. 데스크톱 기준 6페이지 화면 구현
3. 각 페이지를 같은 제품처럼 보이게 통일
4. 샘플 2의 분위기를 유지하되, 해커톤 데모에 맞게 브랜딩 정리

---

# 1. 가장 먼저 반영할 브랜딩 수정

## 1-1. 삭제
- 모든 화면/푸터/소개문/배지에서 **`행정안전부` 문구 제거**
- 정부 공식 서비스처럼 보이는 표현 제거

## 1-2. 대체 표기
다음 중 하나를 공통 사용한다.

- `TownPulse`
- `TownPulse Team`
- `Hackathon Demo`
- `충북 마을생존 AI 의사결정 플랫폼`

## 1-3. 푸터 고정 문구
```txt
© 2026 TownPulse Team. Hackathon Demo.
```

# 2. 제품이 주어야 하는 인상

이 UI는 아래 4개의 인상을 동시에 줘야 한다.

1. **공무원이 봐도 신뢰할 수 있는 정책형 도구**
2. **복잡한 데이터가 빠르게 읽히는 구조**
3. **심플하지만 심심하지 않은 화면**
4. **해커톤 데모치고 완성도 높아 보이는 제품**

즉,
- 지나치게 마케팅 랜딩처럼 화려하면 안 되고
- 너무 공공포털처럼 구식이어도 안 되고
- 너무 스타트업 SaaS처럼 장식적이어도 안 된다.

정답은 다음 사이 어딘가다.

**공공정책용 대시보드 70% + 현대적인 SaaS 30%**

# 4. 디자인 시스템

## 4-1. 컬러 팔레트

### 기본 배경
- `#F6F8FB` : 앱 전체 배경
- `#FFFFFF` : 카드 / 패널 배경

### 브랜드 / 주요 액션
- `#1F4E79` : 메인 네이비
- `#2F6B95` : 보조 네이비
- `#5B86A9` : 약한 보조 톤

### 보조 포인트
- `#3C7A89` : muted teal
- `#D5A64B` : soft amber

### 텍스트
- `#14202B` : 제목
- `#304254` : 본문
- `#66788A` : 보조 텍스트
- `#94A3B8` : 비활성 / 캡션

### 상태 컬러
- 위험: `#C65B57` 배경 연톤 `#FCECEC`
- 주의: `#C08A2E` 배경 연톤 `#FFF6E3`
- 안전: `#4E8B68` 배경 연톤 `#EAF6EE`
- 정보: `#4E78A0` 배경 연톤 `#EDF4FB`

## 4-2. 타이포 시스템

### 폰트
- Pretendard 또는 Inter/Pretendard 계열 우선
- 한글 가독성 최우선

### 크기 제안
- Display Hero: `40px / 48px / 700`
- Page Title: `28px / 36px / 700`
- Section Title: `20px / 28px / 700`
- Card Title: `16px / 24px / 600`
- Body: `14px / 22px / 400~500`
- Caption: `12px / 18px / 500`
- KPI Number: `28~36px / 1.1 / 700`

### 원칙
- 제목은 짧고 강하게
- 설명은 2줄 이내 위주
- 숫자는 다른 텍스트보다 확실히 크게
- 한 카드 안에 텍스트 레벨은 3단계 이내로 제한

## 4-3. 간격 시스템
8px 베이스 사용

- 4px: 아주 작은 간격
- 8px: 배지/텍스트 내부
- 12px: 카드 내부 소간격
- 16px: 기본 카드 패딩
- 20px: 컴포넌트 간 소간격
- 24px: 주요 카드 패딩
- 32px: 섹션 내 큰 간격
- 40px: 섹션 간 간격
- 64px: 페이지 블록 간 간격

## 4-4. 보더 / 라운드 / 섀도우
- 카드 라운드: `16px`
- 작은 카드 라운드: `12px`
- 입력/배지 라운드: `9999px` 또는 `10px`
- 보더: `1px solid #E5ECF3`
- 섀도우: 거의 없음 또는 매우 약하게만

# 5. 레이아웃 시스템

## 5-1. 데스크톱 기준 캔버스
- 기준 해상도: `1440px` 폭
- 콘텐츠 최대 너비: `1280px`
- 좌우 여백: `32px`

## 5-2. 공통 앱 구조
랜딩을 제외한 내부 페이지는 아래 구조를 따른다.

```txt
┌──────────────────────────────────────────────┐
│ Header 64px                                 │
├──────────────┬───────────────────────────────┤
│ Sidebar 220  │ Main Content                  │
│              │                               │
│              │                               │
└──────────────┴───────────────────────────────┘
```

## 5-3. Header 규칙
- 높이: `64px`
- 좌측: 로고 `TownPulse`
- 중앙/좌측: 페이지 제목 또는 breadcrumb 선택 가능
- 우측: `Hackathon Demo` 배지, 테마 토글, `데모 읽기 전용` 배지
- 헤더는 흰색 배경 + 하단 얇은 구분선

## 5-4. Sidebar 규칙
- 폭: `220px`
- 메뉴 간 간격 충분히
- 현재 페이지 active 상태 명확히
- 아이콘 + 텍스트 조합
- 메뉴 수는 5개 정도로 제한

## 5-5. Main Content 규칙
- 상단/하단 패딩: `24~32px`
- 카드와 카드 사이 gap: `20~24px`
- 2열 이상일 때는 한 열이 과도하게 좁아지지 않도록 `min-width` 고려

# 6. 공통 컴포넌트 상세 규칙

## 6-1. KPI 카드
구조:
```txt
[아이콘] 라벨
큰 숫자
보조 설명 / 기준 시점
```

규칙:
- 높이 120~140px
- 숫자 가장 강조
- 라벨은 1줄
- 배경 흰색
- 필요 시 우측 상단에 작은 상태 배지 가능

## 6-2. 상태 배지
- 텍스트 포함 필수: `위험`, `주의`, `안전`
- 아이콘 없이도 의미가 통하게
- pill 스타일
- 크기 예시: `height 28px`, `padding 0 10px`

## 6-3. TOP5 카드
한 줄 구조 예시:
```txt
[#1] 가덕면  | 청주시 상당구
TVI 18 / 100   [위험]
██████░░░░  인구감소 지속 · 정류장 접근 취약
```

규칙:
- 카드 높이 88~104px
- 순위는 시각적으로 구분
- TVI 숫자는 굵게
- 사유는 1줄까지만
- 카드 5개가 세로로 나열될 때 답답하지 않게 gap 12~16px

## 6-4. 지도 패널
- 진짜 지도 라이브러리 삽입 전에는 플레이스홀더라도 `행정구역 지도 패널`처럼 보이게
- 지도 영역 높이 320~440px
- 상단에 범례 + 기준시점 노출
- 너무 화려한 맵 스타일 금지

## 6-5. AI 설명 패널
- 채팅 금지
- 문단형 설명 패널
- 좌측 카드 목록 / 우측 설명영역 구조 권장
- 설명 패널은 읽기 쉬운 line-height 유지
- 600자 이내 느낌의 짧은 정책 설명이 어울리게 디자인

## 6-6. 리포트 프리뷰 카드
- 실제 PDF 에디터처럼 보이게 하지 말고, 미리보기 요약 패널처럼 보이게
- 썸네일 1~3개 또는 표지형 카드 가능
- CTA는 공격적이지 않게

# 7. MVP 제약사항 (디자인에도 반영)

## 7-1. 데모 읽기 전용
- 화면 어딘가에 항상 `데모 읽기 전용` 또는 `Hackathon Demo` 배지 표기
- 쓰기 액션은 최소화
- 버튼은 `보기`, `확인`, `체험` 위주

## 7-2. 자유채팅 금지
- 입력창 / 전송 버튼 / 대화 버블 / 질문 입력 UI 금지
- AI는 설명 카드와 요약 문단으로만 표현

## 7-3. 스냅샷 기반 문구 사용
추천 문구:
- `최신 스냅샷 기준`
- `월별 집계 기반`
- `사전 생성 데이터 예시`
- `데모 읽기 전용`

## 7-4. TVI 해석 문구
아래 문구는 대시보드/지도/상세 어디엔가 반복적으로 들어가도 좋다.

```txt
TVI는 낮을수록 위험합니다.
인구 70 · 빈집 20 · 교통 10 가중치 기준
```

# 페이지 1. 랜딩페이지 (`/`)

## 목적
- 처음 방문자가 5초 안에 서비스 목적을 이해
- 데모 체험 진입
- 해커톤 데모 브랜딩 인지

## 레이아웃 구조
```txt
[Top Nav]

[Hero: 좌측 카피 / 우측 제품 미리보기]

[Stats 4 cards]

[Feature 4 cards]

[3-step How it works]

[CTA band]

[Footer]
```

## 상세 배치
### A. Top Nav (높이 72px)
좌측:
- TownPulse 로고

우측:
- 서비스 소개
- 주요 기능
- 활용 방식
- 데모 체험 버튼
- 기관 로그인 버튼

### B. Hero (2컬럼, 55:45)
좌측:
- 배지: `Hackathon Demo`
- 큰 제목: `충북 228개 마을 소멸위험, TVI로 한눈에 진단`
- 2줄 설명
- 버튼 2개
- 작은 보조 배지: `데모는 읽기 전용`

우측:
- 미니 대시보드 프리뷰 카드
- 내부 구성: 작은 KPI 2개 + 미니 지도 + TOP5 축약 리스트

### C. Stats 4 cards
- 분석 대상 마을
- 위험 등급 마을
- 교통 공백 후보
- AI 처방 사전 생성

### D. Feature 4 cards
아이콘 + 제목 + 2줄 설명
- TVI 위험 지수
- 소멸위험 지도
- AI 처방 설명
- PDF 리포트

### E. How it works
가로 3단계
1. 마을 선택
2. 위험 원인 확인
3. 처방 설명·리포트

숫자 pill + 짧은 설명 구조

### F. CTA Band
배경은 흰색 카드 또는 연한 네이비 틴트
- 제목 1줄
- 보조 설명
- 버튼 2개

### G. Footer
```txt
© 2026 TownPulse Team. Hackathon Demo.
```

## 구현 포인트
- 랜딩은 앱 내부보다 조금 더 여백이 넓고 친절한 설명형 구조
- 그러나 여전히 공공정책 제품처럼 보여야 함
- 마케팅 랜딩처럼 과장된 일러스트 금지

# 페이지 2. 통합 대시보드 (`/dashboard`)

## 목적
- 첫 실사용 화면
- 지역 현황, 위험 마을, AI 요약을 한 번에 보여줌

## 전체 와이어프레임
```txt
[Header]
[Sidebar]

[Page Title + 기준시점 + 읽기전용 배지]

[KPI 4 cards]

[좌: 지역 현황 + 지도 프리뷰] [우: 위험 마을 TOP5]

[하단: AI 처방 설명 / 최근 요약]
```

## 상세 배치
### A. 상단 메타 영역
- 페이지 타이틀: `통합 대시보드`
- 서브텍스트: `충북 마을 생존위험 현황 요약`
- 우측 배지: `최신 스냅샷 기준`, `데모 읽기 전용`

### B. KPI 4 cards (4열)
각 카드 높이 128px 내외
- 분석 마을
- 위험 등급
- 빈집 취약
- 교통 공백

### C. 중단 2컬럼
좌측 60%
- 지역 현황 요약 카드
- 지도 프리뷰 카드
- 지도 하단 짧은 범례

우측 40%
- 위험 마을 TOP5 패널
- 상단에 `TVI는 낮을수록 위험` 캡션
- 5개 카드 세로 정렬

### D. 하단
- 좌측: 최근 AI 인사이트 / 주요 알림
- 우측: 1순위 처방 설명 카드

## 구현 포인트
- 이 페이지는 심사위원이 가장 오래 볼 가능성이 높음
- 숫자와 등급이 가장 먼저 읽혀야 함
- 시선 흐름은 `KPI → 지도 → TOP5 → AI 설명`

# 페이지 3. 소멸위험 지도 (`/map`)

## 목적
- 지역별 위험도 공간 분포 확인
- 위험 지역 탐색

## 와이어프레임
```txt
[Header]
[Sidebar]

[Page Title + Filter Bar]

[큰 지도 패널 70%] [우측 요약 패널 30%]

[하단 보조 인사이트 카드들]
```

## 상세 배치
### A. Filter Bar
- 시군 선택 드롭다운
- 위험등급 필터 chip
- 기준 시점 표기

### B. 지도 패널
- 높이 520px 내외
- 제목: `소멸위험 지도`
- 우측 상단에 범례
- 맵 위 클릭 포인트 스타일 절제

### C. 우측 요약 패널
- 선택 지역명
- TVI 점수
- 위험등급 배지
- 인구 / 빈집 / 교통 / 청년유출 요약칩
- 작은 코멘트 2줄

### D. 하단 인사이트 카드
3개 정도 추천
- 위험 원인 상위 신호
- 교통 공백 후보 지역
- 최근 변화 요약

## 구현 포인트
- 지도만 크게 보여주고 끝내지 말고, 오른쪽 카드가 함께 있어야 함
- 공간 정보 + 정책 요약이 함께 보이는 구조

---

# 페이지 4. 마을 상세 리포트 (`/village/[id]`)

## 목적
- 한 마을의 상황을 자세히 읽는 분석 페이지

## 와이어프레임
```txt
[Header]
[Sidebar]

[Village Hero Card]

[Metric Grid 2x3]

[좌: 위험 원인 분석] [우: 악순환/정책 포인트]

[하단: AI 처방 미리보기]
```

## 상세 배치
### A. Hero Card
- 마을명 / 시군명
- TVI 점수 크게
- 위험 배지
- 기준 스냅샷
- 짧은 요약 문장

### B. Metric Grid
6개 카드 권장
- 인구감소
- 빈집 비율
- 고령화율
- 청년 순이동
- 정류장 접근성
- 버스 배차 간격

### C. 하단 2컬럼
좌측:
- 주요 위험 신호
- 무엇이 문제인지 bullet 3~4개

우측:
- 정책 검토 포인트
- 악순환 구조 요약
- 단기 대응 포인트

### D. 맨 아래
- AI 처방 미리보기 카드 1개
- `자세한 설명 보기` 정도의 CTA 가능

## 구현 포인트
- 리서치 문서처럼 너무 빽빽하지 않게 카드 중심 유지
- 숫자와 해석이 같이 보여야 함

---

# 페이지 5. AI 처방 (`/prescription`)

## 목적
- 추천 정책을 가장 보기 좋게 설명하는 페이지

## 와이어프레임
```txt
[Header]
[Sidebar]

[Page Hero: 대상 지역 + 상태 배지]

[좌: 처방 랭킹 카드 리스트 40%] [우: AI 설명 패널 60%]
```

## 상세 배치
### A. Hero
- 제목: `AI 처방 추천`
- 대상 지역명
- 기준 시점
- 배지: `설명형 데모`, `읽기 전용`

### B. 좌측 랭킹 카드
- 1순위 처방
- 2순위 처방
- 3순위 처방

각 카드 내용:
- 처방명
- 예상 효과
- 관련 위험 신호
- 예산/기대효과 요약(있으면)

### C. 우측 설명 패널
- 제목: `AI 처방 설명`
- 상단에 요약 배지
- 본문은 문단형 텍스트
- 긴 카드 또는 패널 형태

## 구현 포인트
- 절대 채팅 같으면 안 됨
- 보고서 읽는 듯한 안정감 필요
- 1순위 카드는 약간 더 강조 가능

---

# 페이지 6. 리포트 생성 (`/report`)

## 목적
- 어떤 결과물이 나가는지 보여주는 보고용 페이지

## 와이어프레임
```txt
[Header]
[Sidebar]

[Page Title + 기준시점]

[좌: 포함 항목 패널 35%] [우: 리포트 미리보기 65%]

[하단 액션 바]
```

## 상세 배치
### A. 좌측 패널
카드 목록 형태
- 현황 요약
- TVI 분석
- 위험 원인
- AI 처방
- 예산 요약

체크 UI처럼 보일 수 있지만, 데모에서는 완전한 편집 폼처럼 보이지 않게 디자인

### B. 우측 미리보기
- 리포트 표지 카드
- 내부 섹션 썸네일 2~3개
- 요약 문장 2줄

### C. 하단 액션 바
- `PDF 리포트 보기`
- `리포트 요약 확인`

## 구현 포인트
- 진짜 문서 작성툴처럼 보이기보다, 결과물 안내 화면처럼
- 데모 제품의 마지막 단계로 정돈된 인상
