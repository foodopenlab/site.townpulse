> 팀 성격: **해커톤 출품 준비 팀**

> 구현 범위: **데스크톱 우선 6페이지 + 모바일 반응형 대응**

## 0. 가장 먼저 반영할 변경사항

### 0-1. 브랜딩/표기 변경
- 기존 시안/화면 하단에 들어간 **"행정안전부" 문구는 모두 제거**한다.
- 본 프로젝트는 행정안전부 공식 서비스가 아니라, **해커톤 출품 준비 중인 팀 `TownPulse`의 데모 제품**이다.
- 따라서 브랜딩/푸터/소개문/배지/메타 문구에서 **공식 기관 서비스처럼 오해될 수 있는 표현을 쓰지 않는다.**
- 권장 표기:
  - `TownPulse`
  - `TownPulse Team`
  - `Hackathon Demo`
  - `충북 마을생존 AI 의사결정 플랫폼`

### 0-2. 푸터 교체 규칙
기존 하단 표기에서 기관명이나 정부 부처명이 있다면 제거하고 아래 중 하나로 교체한다.

```txt
© 2026 TownPulse Team. Hackathon Demo.
```

또는

```txt
TownPulse | Hackathon Demo for Local Policy Decision Support
```

## 1. 프로젝트 배경

TownPulse는 충북 지역 마을의 소멸위험을 **TVI(Town Vitality Index)** 중심으로 진단하고, 공공데이터 기반으로 위험 원인을 요약하며, AI 처방 설명과 리포트 생성을 제공하는 **데이터 기반 정책 보조 서비스**다.

이번 구현은 단순한 와이어프레임이 아니라, **해커톤 심사에서 바로 보여줄 수 있는 수준의 중충실도 UI**를 목표로 한다.

다만 MVP 단계이므로 실제 구현은 아래 원칙을 반드시 따른다.

- 공무원이 사용해도 될 정도로 **가독성 높고 신뢰감 있는 UI**
- 그러나 기존 MVP보다 **덜 딱딱하고 덜 지루한 화면**
- 과한 그래디언트, 번쩍이는 효과, 과장된 애니메이션 금지
- **샘플 2의 레이아웃 톤**을 기준으로 확장
- **데스크톱 우선**, 모바일은 반응형으로만 대응

## 2. 핵심 톤앤매너

### 2-1. 시각 스타일
- **Flat / Minimal / Clean / Readable**
- 라이트 테마 기본
- 컬러는 **네이비 + 뮤트 틸 + 소프트 앰버** 중심
- 카드 UI 적극 사용
- 그림자 최소화 또는 제거
- 그라데이션 사용 금지
- 둥근 모서리 사용 가능하되 과하지 않게 (`rounded-xl` 정도)
- 구분선은 얇고 정돈되게
- 여백을 충분히 줘서 공공/행정 서비스 특유의 답답함을 줄일 것

### 2-2. 사용자 인상
사용자가 처음 봤을 때 아래 인상이 들어야 한다.

- "공공/정책용 도구라 믿을 만하다"
- "정보가 빨리 읽힌다"
- "딱딱하긴 한데 촌스럽지는 않다"
- "해커톤 데모치고 완성도가 높다"

### 2-3. 피해야 할 느낌
- 정부 포털 같은 낡은 레이아웃
- 스타트업 랜딩처럼 과도하게 마케팅 중심인 톤
- 실시간 관제실처럼 과도하게 붉고 자극적인 대시보드
- 자유 입력형 AI 채팅 서비스처럼 보이는 구성

## 3. MVP 제약사항 (반드시 준수)

### 3-1. 데모 모드 제약
- 데모는 **읽기 전용**이다.
- 저장, 수정, 생성, 제출, 업로드처럼 보이는 액션은 최소화하거나 비활성화한다.
- 화면상에서 데모 사용자에게는 `데모 읽기 전용` 배지를 명확히 보여준다.

### 3-2. AI 기능 제약
- **자유채팅 UI 금지**
- 챗봇 입력창, 대화창, 메시지 송수신형 인터페이스 금지
- AI는 **처방 설명/요약/근거 제시** 형태로만 표현한다.
- 즉, UI는 `AI 처방 설명 카드`, `스트리밍 설명 영역`, `처방 요약 패널`처럼 보여야 한다.

### 3-3. 데이터 성격
- 실시간 라이브 데이터처럼 과장하지 않는다.
- `최신 스냅샷 기준`, `월별 집계 기준`, `사전 생성 데이터 기준` 같은 문구를 사용한다.
- 랜딩/대시보드/리포트 전반에서 **snapshot-driven product**라는 인상이 나야 한다.

### 3-4. TVI 고정 규칙
TVI는 아래 가중치 체계를 설명용으로 유지한다.

- 인구: 70
- 빈집: 20
- 교통: 10

가중치 슬라이더, 사용자 편집, 커스터마이징 기능은 넣지 않는다.

## 4. 공통 레이아웃 규칙

### 4-1. 데스크톱 레이아웃
- 상단 헤더 + 좌측 사이드바 + 메인 콘텐츠 구조 사용
- 랜딩페이지만 예외적으로 사이드바 없이 전체폭 구조 허용
- 콘텐츠 최대 폭은 너무 넓게 퍼지지 않도록 제한 (`max-width` 관리)

### 4-2. 반응형 원칙
- 모바일 앱은 MVP에서 미구현이므로 **모바일 전용 화면을 새로 설계하지 않는다**
- 대신 데스크톱 UI가 자연스럽게 축소되도록 반응형 처리한다.
- 브레이크포인트 예시:
  - `lg`: 데스크톱 기준 완성형
  - `md`: 2열 → 1열 일부 축소
  - `sm`: 카드 스택 구조

### 4-3. 공통 헤더 요소
- 로고: `TownPulse`
- 보조 배지: `Hackathon Demo`
- 우측: 테마 토글, 데모 읽기 전용 배지(로그인 전/데모 상태에서)

### 4-4. 공통 사이드바 메뉴
다음 메뉴 체계를 기본으로 사용한다.

1. 대시보드
2. 소멸위험 지도
3. 마을 리포트
4. AI 처방
5. 리포트 생성

## 5. 공통 컴포넌트 규칙

### 5-1. KPI 카드
- 숫자 크게
- 라벨은 짧고 명확하게
- 서브텍스트로 기준 시점 표시 가능
- 아이콘은 단색 라인 아이콘 위주

### 5-2. 위험 배지
- `위험` / `주의` / `안전`
- 컬러만으로 의미 전달하지 말고 텍스트 배지도 함께 표시

### 5-3. TOP5 카드
반드시 아래 정보 포함
- 순위
- 마을명
- 시군명
- TVI 점수 `/100`
- 위험 등급 배지
- 얇은 진행바
- 한 줄 위험 사유

예시 사유
- `인구감소 지속`
- `빈집 비율 높음`
- `정류장 접근 취약`
- `청년 순유출`
- `고령화 비율 높음`

### 5-4. 보조 설명 문구
대시보드/지도/마을상세 어딘가에 아래 문구를 자연스럽게 표시한다.

```txt
TVI는 낮을수록 위험합니다.
인구 70 · 빈집 20 · 교통 10 가중치 기준
```

### 5-5. AI 처방 카드
- 1순위 / 2순위 / 3순위 구분 가능
- 1순위 카드는 시각적 강조 가능
- 예산/기대효과/적용배경 정도는 카드 안에 요약 가능
- 자유 입력창은 절대 넣지 않는다

## 페이지 1. 랜딩페이지 (`/`)

### 목적
- 로그인 없이도 서비스 목적을 이해시킨다.
- 해커톤 데모임을 자연스럽게 보여준다.
- 사용자를 `데모 체험` 또는 `기관 로그인`으로 유도한다.

### 핵심 섹션
1. 상단 네비게이션
2. 히어로 섹션
3. 핵심 통계 카드 4개
4. 주요 기능 소개 4개
5. 사용 흐름 3단계
6. 하단 CTA
7. 푸터

### 꼭 들어갈 문구 방향
- 메인 카피 예시:
  - `충북 228개 마을 소멸위험, TVI로 한눈에 진단`
- 서브 카피 예시:
  - `인구감소·빈집·교통 데이터를 바탕으로 마을 위험도를 읽고, 정책 판단에 필요한 AI 처방 설명을 빠르게 확인할 수 있습니다.`
- 보조 배지:
  - `Hackathon Demo`
  - `데모는 읽기 전용`

### 우측 프리뷰
- 반드시 **미니 대시보드 미리보기**로 구성
- 채팅형 UI 금지
- 작은 지도 + TOP5 리스트 + KPI 카드 조합 권장

### CTA 버튼
- `데모 체험`
- `기관 로그인`

### 푸터
- `행정안전부` 문구 금지
- `© 2026 TownPulse Team. Hackathon Demo.` 사용

## 페이지 2. 통합 대시보드 (`/dashboard`)

### 목적
- 충북 전체 또는 선택 지역의 위험 현황을 한 번에 읽게 한다.
- 심사위원/공무원/관계자가 첫 화면에서 제품 가치를 이해하게 한다.

### 구성
#### 상단 KPI 4개
- `분석 마을`
- `위험 등급`
- `빈집 취약`
- `교통 공백`

#### 메인 좌측
- 지역 현황 요약 카드
- TVI 기준 지도 프리뷰
- 기준 시점/스냅샷 정보

#### 메인 우측
- `위험 마을 TOP 5`
- 각 카드에 순위 / 마을 / 시군 / TVI / 배지 / 사유

#### 하단
- `AI 처방 설명`
- `빠른 보기` 또는 `최근 요약`

### 반드시 반영할 요소
- `데모 읽기 전용` 배지
- `최신 스냅샷 기준` 문구
- `TVI는 낮을수록 위험` 설명
- `인구 70 · 빈집 20 · 교통 10` 설명

### 금지
- 실시간 라이브 점멸 효과
- 채팅 입력창
- 편집 버튼 과다 노출

## 페이지 3. 소멸위험 지도 (`/map`)

### 목적
- 공간적으로 어느 지역이 더 위험한지 직관적으로 보여준다.
- 전체 현황에서 특정 마을로 진입하는 중심 페이지 역할을 한다.

### 구성
#### 상단
- 페이지 제목
- 필터 영역 (시군 선택, 위험등급 필터 정도)
- 기준 시점 표기

#### 본문 좌측
- 대형 지도 영역
- TVI 기반 컬러 단계 표시
- hover/click 시 요약 툴팁 또는 우측 카드 연동

#### 본문 우측
- 선택 지역 요약 카드
- 주요 지표 미니 카드
- 교통/빈집/고령화/청년유출 요약

### 지도 시각화 방향
- Leaflet 스타일을 염두에 둔 단정한 행정구역 뷰
- 과한 3D, 위성맵 느낌 금지
- 색상은 위험/주의/안전 3단계 중심

### 하단 또는 우측 보조영역
- `위험 원인 상위 신호`
- `인구감소`, `빈집`, `교통 공백`, `청년 순유출` 등 요약 태그

## 페이지 4. 마을 상세 리포트 (`/village/[id]`)

### 목적
- 특정 마을 하나를 깊이 있게 분석하는 화면
- 실제 정책 검토/보고 전에 읽는 상세 페이지 느낌

### 상단 헤더 영역
- 마을명
- 시군명
- TVI 점수
- 위험 등급 배지
- 기준 스냅샷 시점

### 본문 1열/2열 구성
#### 좌측 핵심 진단
- TVI 요약 카드
- 인구감소
- 빈집 비율
- 고령화율
- 정류장 접근성
- 버스 배차 간격

#### 우측 분석 요약
- 위험 원인 한눈에 보기
- 악순환 구조 요약
- 전월/기준월 대비 변화 요약(가능하면)

### 추가 섹션
- `주요 위험 신호`
- `정책 검토 포인트`
- `AI 처방 미리보기`

### 디자인 포인트
- 숫자/그래프/막대/배지 조합
- 너무 리서치 문서처럼 빽빽하지 않게 카드와 섹션 분리

---

## 페이지 5. AI 처방 (`/prescription`)

### 목적
- 특정 마을 또는 지역에 대해 AI가 추천하는 정책 방향을 보여준다.
- 단, 채팅 제품처럼 보이지 않아야 한다.

### 상단
- 페이지 제목
- 대상 마을/지역 표시
- 읽기 전용 또는 설명형 배지

### 메인 구성
#### 좌측
- 1순위 / 2순위 / 3순위 처방 카드
- 각 카드에 아래 정보 포함 가능
  - 처방명
  - 기대 효과
  - 관련 근거
  - 예상 예산(있다면)
  - TVI 개선 기대값(있다면)

#### 우측
- `AI 처방 설명` 스트리밍형 패널
- 문단형 설명 UI
- 행정 보고 톤의 정돈된 텍스트

### 절대 금지
- 메시지 입력창
- 전송 버튼
- 사람과 AI가 대화하는 버블형 구조

### 추천 표현
- `AI가 사전 생성된 처방 근거를 설명합니다`
- `최신 스냅샷 기준 분석 요약`

---

## 페이지 6. 리포트 생성 (`/report`)

### 목적
- 보고용 결과물을 정리해 내보내는 마지막 단계
- 실제 생성/다운로드보다도 "무슨 항목이 포함되는지"가 명확해야 한다.

### 화면 구성
#### 상단
- 페이지 제목
- 대상 지역/마을
- 기준 시점

#### 좌측
- 포함 항목 선택 카드처럼 보이는 영역
  - 현황 요약
  - TVI 분석
  - 위험 원인
  - AI 처방
  - 예산 요약

## 10. 공통 구현 디테일

### 버튼 스타일
- Primary: 네이비 계열
- Secondary: 화이트 배경 + 네이비 보더
- Danger 버튼 스타일은 가급적 쓰지 않음

### 카드 스타일
- 배경 흰색
- 얇은 보더
- 내부 패딩 넉넉하게
- 그림자 거의 없음

### 타이포 방향
- 제목은 선명하고 강하게
- 본문은 너무 작지 않게
- 표/배지/설명 텍스트 간 위계 분명히

### 아이콘 방향
- 단색 또는 2톤 이내
- 과하게 귀엽지 않게
- 공공정책 제품에 맞는 차분한 라인 아이콘 사용

## 12. Cursor에게 최종 작업 요청

아래 기준으로 바로 구현 진행한다.

1. **샘플 2 디자인 톤을 기준으로 6페이지 전체를 확장 구현**한다.
2. 기존 시안/푸터/카피에서 **`행정안전부` 관련 표기를 모두 제거**한다.
3. 본 프로젝트는 **공식 기관 서비스가 아니라 `TownPulse` 해커톤 데모 팀 프로젝트**라는 점을 반영한다.
4. 모든 화면은 **데스크톱 우선**으로 구현하되, 모바일에서는 카드 스택 기반으로 자연스럽게 반응형 처리한다.
5. 자유채팅 UI를 만들지 말고, AI는 **설명/근거/요약 패널**로만 구현한다.
6. 모든 페이지는 **가독성 높은 공공정책용 UI**이되, 기존 MVP보다 덜 딱딱하고 덜 심심하게 다듬는다.
7. 구현 후에는 공통 컴포넌트화가 가능하도록 구조를 정리한다.
